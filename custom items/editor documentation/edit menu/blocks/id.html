Every custom block created by this plug-in has a distinct numerical ID. This is an integer between 1 and 159, which is why you can have at most 159 custom blocks.

All custom blocks will be represented by mushroom blocks in-game. Mushroom blocks have different textures, depending on whether there is an adjacent block on each side. For instance, a mushroom block can show texture A when it is not surrounded by any blocks and it can show texture B when a block is placed on top of it. This allows 2^6 = 64 textures per mushroom block. Minecraft has 3 such mushroom blocks: red mushroom blocks, brown mushroom blocks, and mushroom stems. This would allow 3 * 64 = 192 textures in total. Only 33 of these textures will be shown by natural mushroom blocks; the other 159 will be used by this plug-in to represent custom blocks.

The mechanic explained above is not every useful on its own: custom blocks would only show their desired texture when there are solid blocks placed adjacent to its right sides. But, this plug-in can force mushroom blocks to stay in the desired state with the desired texture.

The numerical ID of a custom block determines which mushroom block type will be used to represent the custom block, and which 'sides' of the block are 'occupied'. The table below shows which numerical ID is mapped to which mushroom block type and on which sides it would have adjacent blocks. This information can be useful for advanced users that would like to create custom blocks through other means than simply placing it.

The following example demonstrates how to read this table: If you have a custom block with numertical ID 1, it will be represented by a brown mushroom block, and it will take the texture of a brown mushroom block that is adjacent to a block on its west side and is not adjacent to any other blocks. (But the plug-in will make sure that you don't need to worry about the presence/absence of adjacent blocks.)
