The upgrade result menu can be used to create or edit upgrade results. Upgrade results can add 0 or more upgrades to an ingredient of the recipe. Furthermore, upgrade results can optionally repair the item, remove previous enchantments/upgrades, or turn the ingredient into something else. The upgrade result menu should look like this:

Upgrades (optional)

Click the Upgrades... button to choose which upgrades should be added to the ingredient. You can choose 0 or more upgrades.

Repair (optional)

How much the ingredient should be repaired when it is upgraded. Use a positive percentage to repair the item, use a negative percentage to damage the ingredient, and use 0 to keep the durability the same. This is ignored for unbreakable items (and simple items that don't have durability).

New type (optional)

The new item type of the ingredient. You can use this to turn the ingredient into a different item, while keeping properties like enchantments or durability. By default, the new type is null, which will cause the ingredient to keep its own item type.

Keep old upgrades

Whether the ingredient should keep its existing upgrades. If you uncheck this, the ingredient will lose all of its previous upgrades (including enchantments). This can be useful if you want some 'exclusive' upgrades.

Keep old enchantments

Whether the ingredient should keep its non-KCI enchantments (the enchantments that were for instance added by an enchanting table). If you uncheck this, the ingredient will only keep the enchantments given by previous upgrades (if Keep old enchantments is still checked) and the default enchantments (only applicable to custom items).

Choose ingredient (required)

Click the Choose ingredient... to choose which ingredient should be upgraded.
